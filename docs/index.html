Per-scene output, recording, and monitoring presets for OBS Studio

Switch scenes and automatically apply that scene's resolution, FPS, recording folder, format, tracks, bitrate, and monitoring device. A global mute control lets you stop hearing monitored audio without changing the recording.

Per-scene presets Scene duplication friendly Monitoring-safe mute Compact OBS dock

What It Does

OBS output and recording settings are normally profile-wide. Auto Resize Output lets each scene store the settings it should apply when that scene becomes active.

Presets are stored in the scene source's private_settings, so they are saved with the scene collection and copied when a scene is duplicated.

Scene presets apply when entering a scene

The plugin does not automatically restore values from a previous scene. If Scene B does not override FPS, the current FPS remains whatever it already was. To make a scene use a value, explicitly enable and set that override for the scene.

Close OBS before replacing the DLL. On Windows, install the plugin files here:

Enable Overrides

  1. Select a scene in Editing scene.
  2. Enable Enable per-scene overrides for this scene.
  3. Check only the fields this scene should override.
  4. Change values. The preset is saved as you edit.

OBS cannot change video settings while outputs are active

Resolution and FPS changes require the OBS video pipeline to be idle. If recording, streaming, or virtual camera output is active, OBS rejects the change. The optional restart-recording setting can stop recording, apply video changes, and start a new recording when no stream or virtual camera output is active.

Recording

Recording settings are staged for the next recording. They do not rewrite an already-running recording output.

Actions

  • Copy from current OBS settings fills the form with current OBS values without automatically enabling any overrides.
  • Apply now saves and applies immediately when the edited scene is also the current program scene.

Monitoring Vs Recording

OBS audio has two separate paths. The output path goes to recording and streaming. The monitoring path is playback for you to hear. Auto Resize Output only changes the monitoring path for monitoring features.

The sound must pass through OBS monitoring

Open OBS Advanced Audio Properties and set the source's Audio Monitoring to Monitor and Output or Monitor Only. If the operating system is playing the sound directly to your headphones, OBS monitoring controls cannot selectively mute it.

Per-Scene Monitoring Device

Enable Monitoring device and choose where OBS should play monitored audio when the scene becomes active. This can change while recording because it does not touch the encoder path.

Mute To Me

The top button globally stops you from hearing OBS-monitored audio. Press it again to hear monitoring again.

  • Windows: mutes OBS playback sessions on the current monitoring endpoint and restores their previous mute state when disabled.
  • Other platforms: routes OBS monitoring to a silent sentinel device as a fallback.
  • Recording content, recorded volume, source volume, and track selection are not changed.
Why not toggle each source's monitoring type?

OBS persists each source's monitoring type when saving a scene collection. If a plugin changed monitoring types for mute, a save while muted could permanently overwrite a user's Monitor and Output setup. This plugin avoids that path.

Limits

  • Resolution and FPS changes require OBS outputs to be idle, unless the plugin restarts recording.
  • Recording bitrate override requires Advanced output mode.
  • Mute to me only controls audio heard through OBS monitoring.
  • Recording folder, format, tracks, and bitrate apply to the next recording, not the current output.

Example Workflows

Game scenes at 1080p60, talk scenes at 1080p30

Enable output resolution and FPS overrides on each scene. Switch to the scene before starting recording, or allow the plugin to restart recording if that workflow is acceptable.

Different folders or formats per scene

Enable recording folder and format overrides for each scene. The next recording uses the active scene's staged values.

Long recordings where you stop listening mid-session

Set sources to Monitor and Output, then use Mute to me. The recording continues unchanged.

FAQ

I clicked Mute to me but still hear sound.

First make sure OBS monitoring is actually producing sound. Then verify the sound is not being played directly by the operating system. The source must use OBS monitoring for the plugin to control what you hear.

Resolution did not change when I switched scenes.

An OBS output is probably active. OBS does not allow video pipeline changes while recording, streaming, or virtual camera output is running.

How do I leave compact dock mode?

Right-click the compact Mute / Hear button and choose Show full settings.

Do presets follow duplicated scenes?

Yes. Presets live in source private settings, which OBS copies during scene duplication and serializes with the scene collection.
