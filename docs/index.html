讓每個 Scene 各自決定輸出與錄製方式

切換 Scene 就自動套用該 Scene 的解析度、FPS、錄製資料夾、格式、音軌、位元率;還能控制「你自己聽到的聲音」而完全不影響錄製。

這個外掛在做什麼

OBS 本身的輸出/錄製設定是全域的:整個設定檔共用一組解析度、FPS、錄製資料夾……。本外掛讓你為每一個 Scene 分別設定這些東西,切到該 Scene 時自動套用。

設定直接存在 Scene source 的 private_settings 裡,因此:

  • 複製 Scene → 設定自動一起複製。
  • 存檔 → 設定隨 Scene Collection 一起存,不需要額外檔案。
設計原則:進入該 Scene 時「套用」,不會自動還原

若 Scene B 沒有勾選某個欄位,該欄位會維持目前的值(也就是上一個 Scene 設過的),而不是回到預設。要讓某個 Scene 用特定值,就在該 Scene 明確勾選並填好它。

安裝 / 開啟面板

把外掛安裝到 OBS 的外掛資料夾(Windows):

檔案位置
外掛 DLL%ProgramData%\obs-studio\plugins\obs-auto-resize-output\bin\64bit\obs-auto-resize-output.dll
語系檔%ProgramData%\obs-studio\plugins\obs-auto-resize-output\data\locale\en-US.ini
  1. 完全關閉 OBS(安裝/更新 DLL 時 OBS 不能開著,否則檔案被鎖住)。
  2. 把 DLL 與 data/ 放到上述位置。
  3. 開啟 OBS,從上方選單 Docks → 勾選 Auto Resize Output,面板就會出現(可拖曳停靠到任意位置)。

面板總覽

面板由上到下分成幾個區塊:

區塊作用
Mute to me 按鈕(最上方)全域一鍵開關:讓你自己聽不到監聽音訊,錄製不受影響。見 這裡。
Show compact mute overlay開啟 always-on-top 小浮窗,只顯示 Mute/Hear 控制;可拖曳到 OBS 上方方便操作。
Editing scene 下拉選擇要編輯哪個 Scene 的設定。預設跟隨目前直播中的 Scene。
Enable per-scene overrides該 Scene 的總開關。關閉時整個 Scene 不套用任何覆寫。
Video 群組基底/輸出解析度、FPS(僅在閒置時生效)。
Recording 群組錄製資料夾、格式、音軌、影像位元率(套用到下一次錄製)。
Audio monitoring 群組該 Scene 的監聽裝置(錄製中也能即時切)。
底部按鈕Copy from current OBS settings 與 Apply now。
面板不會再「搶走」你的選擇

當你手動在下拉選了「別的 Scene」來編輯時,即使你切換了直播 Scene,面板也不會跳走,方便你安心編輯。等你再選回目前直播中的 Scene,就會恢復自動跟隨。

啟用覆寫

  1. 在 Editing scene 選好要設定的 Scene。
  2. 勾選 Enable per-scene overrides for this scene。
  3. 只勾選你真正想覆寫的項目並填值;沒勾的項目會沿用目前狀態。

每個欄位前面都有自己的勾選框 —— 勾了才會被套用。所有欄位上都有滑鼠停留提示(tooltip),會解釋該設定的意義。

影像:解析度 / FPS

設定意義
Base (canvas) resolutionOBS 合成畫布的整體大小(預覽看到的範圍)。
Output (scaled) resolution實際錄下來的像素尺寸 = 清晰度 / 畫面大小。畫布會被縮放到這個尺寸。
FPS (integer)每秒幀數 = 流暢度。30 普通、60 流暢(適合遊戲/動態)。
解析度與 FPS 只能在「閒置時」改變

這是 OBS 影片管線的硬限制:錄製 / 串流 / 虛擬攝影機運行時無法更改解析度或 FPS。閒置時切到該 Scene 才會立即套用;之後按下錄製就會用該設定。

若你勾選了 restart recording to apply…,且當時只有錄製在跑(沒有串流/虛擬攝影機),外掛會「停止錄製 → 套用新解析度 → 重新開始錄製」(會產生一個新檔案)。

清晰度 vs 流暢度 vs 品質:解析度決定畫面有多大/多細、FPS 決定多流暢、位元率(見下)決定壓縮後的品質,三者互相獨立。

錄製:資料夾 / 格式 / 音軌 / 位元率

這一組設定會寫進設定檔,套用到「下一次」開始錄製(不會中途改變正在進行的錄製)。

設定說明
Recording folder該 Scene 的錄影存放資料夾(例如把遊戲與教學分到不同資料夾)。
Format容器格式。mkv 最抗當機;mp4/hybrid_mp4 相容性好、可直接剪輯。容器本身不影響畫質。
Audio tracks要錄哪幾條音軌(可複選)。例如軌 1 = 總混音、軌 2 = 麥克風、軌 3 = 遊戲聲,方便後期分開剪。
Video bitrate (kbps)每秒用多少資料編碼影像 = 品質 / 檔案大小。參考:1080p30 ≈ 8000、1080p60 ≈ 12000、1440p60 ≈ 20000。
位元率僅支援 Advanced 輸出模式

位元率寫入 recordEncoder.json 並把錄製編碼器設為 CBR,只在 OBS 的 Advanced 輸出模式有效。Simple 模式下此欄位會被忽略(並於 log 提示)。面板會顯示目前是 Simple 還是 Advanced 模式。

注意:套用位元率會把錄製編碼器切成 CBR;若你原本用 CQP/CRF,套用後會變成 CBR。

Copy / Apply 按鈕

  • Copy from current OBS settings — 用 OBS目前的值快速帶入起始值(解析度、FPS、錄製資料夾/格式/音軌、位元率、監聽裝置)。它只填值,不會自動勾選任何項目,讓你有個起點再微調。
  • Apply now — 立即儲存並套用。若你正在編輯的 Scene 就是目前直播中的 Scene,會馬上生效;否則會在你切到該 Scene 時才套用。

其實你每次改欄位都會自動儲存,Apply now 主要是用來「立刻套用到現在這一刻」。

音訊監聽:先搞懂「監聽」vs「錄製」

這是理解後面兩個功能的關鍵。OBS 裡,一個音源的聲音有兩條獨立的路:

最重要的前提:只有「經過 OBS 監聽」的聲音才控制得到

上圖的「監聽」這條路,預設是關的。要讓某個音源的聲音經過 OBS 監聽(這樣本外掛才管得到你聽不聽得到),你必須先在 OBS 混音器把該音源設成 Monitor and Output:

混音器右下角 ⋮ → Advanced Audio Properties → 該音源的 Audio Monitoring 欄位選 Monitor and Output。

如果某個聲音是作業系統直接播放到你喇叭/耳機的(例如一般桌面音訊,OBS 只是側錄),那你聽到的是系統的聲音,OBS 的監聽開關管不到它。

每個 Scene 的監聽裝置

在 Audio monitoring 群組勾選 Monitoring device 並選一個裝置。切到該 Scene 時,OBS 的全域監聽裝置就會換成它。

  • 錄製中也能切,且不中斷錄製 —— 監聽和錄製是兩條路。
  • 不會改變錄到的內容或音量。
  • 用途:讓某個 Scene 把監聽導到「你沒在聽的裝置」,等於那個 Scene 你聽不到,但照常錄。

若選的裝置目前不存在(例如拔掉的耳機),面板仍會保留它,設定不會遺失。某些平台不支援監聽時,此區塊會自動變灰。

「Mute to me」一鍵靜音(全域)

面板最上方那顆按鈕。一鍵讓你自己瞬間聽不到所有被監聽的音訊,再按一次恢復 —— 錄製全程不受影響(內容、音量都不變),錄製中也能隨時切。不需要為此複製任何 Scene。Windows 版會靜音 OBS 這個 process 的播放 session(類似 EarTrumpet mute obs64.exe),不會動到錄製路徑。

  • 按一下 → Muted to you — click to hear again:你聽不到了。
  • 再按一下 → 恢復聽得到。
  • 需要最小控制介面時,按 Show compact mute overlay。小浮窗只顯示 Mute / Hear,可拖曳移動,狀態會和 dock 按鈕同步。
典型用法

錄一段長影片時,中途你不想再聽那些聲音(吵、或想耳根清淨),按一下 Mute to me;觀眾/錄影檔完全不受影響。想聽了再按回來。

技術說明:為什麼不用「切換每個來源的監聽類型」?

因為 OBS 會在每次存檔(自動存檔、切換 Scene Collection、關閉前存檔)把每個來源的監聽類型寫進 Scene Collection,而關閉時的存檔發生在外掛能還原之前。若用「切換監聽類型」來靜音,一旦在靜音狀態下存檔/關閉,你原本的 Monitor and Output 設定就會被永久覆寫。

Windows 版改用 OS audio session mute,也就是靜音 OBS 播給你的那個播放 session;解除時會還原原本的 session mute 狀態。非 Windows 平台才使用全域監聽裝置 fallback。

重要限制(務必理解)

  • 解析度 / FPS 無法在錄製/串流/虛擬攝影機運行時更改 —— OBS 架構限制。任何宣稱「錄製中無縫切解析度」的方案都做不到。
  • 位元率僅 Advanced 輸出模式,且套用後錄製編碼器會變成 CBR。
  • 「聽不到但照錄」只對經由 OBS 監聽的聲音有效(音源需設為 Monitor and Output);系統直接播放的聲音 OBS 管不到。
  • 錄製設定(資料夾/格式/音軌/位元率)套用到下一次錄製,不會改變進行中的錄製。
  • UI 目前為英文;多語系為日後里程碑。

實用情境

情境 A:遊戲 Scene 用 1080p60,聊天 Scene 用 1080p30 省資源

  1. 在遊戲 Scene 勾選 Output resolution 1920×1080 + FPS 60。
  2. 在聊天 Scene 勾選 FPS 30。
  3. 錄製開始前切到想要的 Scene,設定就會套用。

情境 B:不同 Scene 存到不同資料夾、不同格式

在各 Scene 分別勾選 Recording folder / Format,下一次錄製就會按該 Scene 的設定走。

情境 C:長時間錄製,中途讓自己耳根清淨

把要錄的音源設成 Monitor and Output → 隨時按 Mute to me 切換你聽不聽得到,錄影不受影響。

疑難排解 / FAQ

我按了 Mute to me,但還是聽得到聲音?

若按鈕沒有維持在 muted 狀態、底部顯示找不到 active OBS playback session,請先讓 OBS monitoring 實際出聲,再按一次 Mute to me。

若按鈕已經顯示 muted 但你仍聽得到,那個聲音多半是系統直接播放的(不是經由 OBS 監聽)。請到混音器 → Advanced Audio Properties,把該音源的 Audio Monitoring 設成 Monitor and Output,Mute to me 才管得到。

切換 Scene 時解析度沒變?

你大概正在錄製或串流。解析度/FPS 只能在閒置時改。要在錄製中換,需勾選該 Scene 的「restart recording to apply」(會產生新檔案,且只在沒有串流/虛擬攝影機時可行)。

位元率欄位沒作用?

確認 OBS 在 Advanced 輸出模式(面板上會顯示目前模式)。Simple 模式不支援單獨設定錄製位元率。

找不到面板?

上方選單 Docks → 勾選 Auto Resize Output。若整個外掛沒載入,確認 DLL 放在正確路徑、且安裝時 OBS 是關閉的。

設定會跟著 Scene 複製嗎?

會。設定存在 Scene 的 private_settings,複製 Scene 時一起複製,存檔時隨 Scene Collection 一起存。
